How To Deploy A CodePen Using Google Pages

Step 1: Linking Codepen to Github

  1. If you do not have one already, create a GitHub account using the same email you used to create your CodePen account. If you need to change/add an email to your CodePen, you can do so here
  2. Log into CodePen using Github.

Step 2: Transfer the CodePen to GitHub

  1. Open up the pen you would like to host.
  2. At the bottom of the screen find the Export button
  3. Select "Save as GitHub Gist"
  4. On your GitHub account, create a new repository
  5. At the bottom of your repository, select "Import Code"
  6. Copy the url from the Gist that you exported, and paste into the url bar in the repository
  7. Wait for the files to import

Step 3: Enable Pages Hosting

  1. In your repository, select Settings from the top bar
  2. In the settings section, select Pages, underneath Code and automation
  3. Underneath Branch, select main using the drop down menu, then click save
  4. It can take some time to set up, but once it does, there should be an Enviroments section in the <code> section of your repository. In this section should be github-pages. Click this.
  5. Click "View Deployment" to view your website

Help, my site doesn't look like it does on CodePen!?!

CodePen does not require linking your html page to your css page in order for stylings to be applied. This is not the case for GitHub Pages. Make sure you have a link to your css page in your <head> tag of your html. CodePen exports the css file as style.css, so your link should look like this:
<link rel="stylesheet" href="style.css">
Including this line in your html file links your html to your css file. You can make this edit by selecting your html file from the code segment of your repository (make sure to click on the file name itself, and not the blue comment section), and then click on the pencil on the top bars of the page to edit your html file.

Help, my javascript isn't working

This could be due to several issues, but a few big ones are as follow:

  1. You have not included a proper link to your js file in your head. The proper way to do this is using the <script> tag. All you need to do to implement this is include the following code in your head:
    <script src="script.js"> </script>
    Keep in mind, script.js is just how Codepen exports the js file. In a more standard setting, the js files would be stored in a separate folder, possibly with more js files, so you would use the following format instead:
    <script src="./js/filename.js"> </script>
  2. There is possibly an error in your javascript, whether it be pathing to linked elements, or just common user errors. Double check that there are no typos, no missing semicolons (;), etc.
  3. Github pages is currently not designed to run server side code, so it is possible that one or more of your functions can not run on Github pages. If your issue is not one of the previous issues, it is most likely this.
